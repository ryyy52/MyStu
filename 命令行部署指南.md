# 命令行部署指南

如果脚本部署遇到问题，您可以使用以下命令行步骤手动部署项目。

## 方法一：使用管理员权限的命令提示符

1. **打开管理员命令提示符**：
   - 按下 `Win + X`，选择「命令提示符(管理员)」或「Windows PowerShell(管理员)」

2. **执行复制命令**：
   ```cmd
   copy "c:\Users\a2423\Desktop\WWEB\target\ecommerce-system.war" "D:\apache-tomcat-8.5.84\webapps\" /Y
   ```

## 方法二：使用PowerShell命令

1. **以管理员身份打开PowerShell**：
   - 按下 `Win + X`，选择「Windows PowerShell(管理员)」

2. **执行复制命令**：
   ```powershell
   Copy-Item -Path "c:\Users\a2423\Desktop\WWEB\target\ecommerce-system.war" -Destination "D:\apache-tomcat-8.5.84\webapps\" -Force
   ```

## 方法三：使用Maven命令部署

确保已在`pom.xml`中配置了Tomcat Maven插件，并且Tomcat管理器已正确设置。

1. **构建并部署**：
   - 正常命令提示符中执行：
   ```cmd
   mvn clean package tomcat7:redeploy
   ```

## 验证部署

部署完成后，请：

1. 确保Tomcat服务已启动
2. 打开浏览器访问：http://localhost:8080/ecommerce-system/

## 常见问题解决

- **权限被拒绝**：确保以管理员身份运行命令行工具
- **文件找不到**：检查路径是否正确，确保WAR包已成功构建
- **Tomcat未响应**：确保Tomcat服务已启动，可尝试重启Tomcat

## 手动确认路径

如果遇到路径问题，可以先执行以下命令验证路径是否正确：

```cmd
rem 检查WAR包是否存在
dir "c:\Users\a2423\Desktop\WWEB\target\ecommerce-system.war"

rem 检查Tomcat目录是否存在
dir "D:\apache-tomcat-8.5.84\webapps"
```
